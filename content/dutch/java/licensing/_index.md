# Stel GroupDocs-licentie Java in – Licentiehandleidingen voor GroupDocs.Merger

In dit hub stel je **groupdocs-licentie java instellen** snel en correct in, of je nu werkt met een licentiebestand, een `InputStream`, of een model met meter‑gebruik. We leiden je door elk licentiescenario, leggen uit waarom juiste licentiëring belangrijk is voor productie‑stabiliteit, en wijzen je op de meest actuele code‑voorbeelden. Door deze handleidingen te volgen vermijd je veelvoorkomende valkuilen zoals “License not found”-fouten en zorg je ervoor dat je document‑verwerkings‑pijplijnen volledig functioneel blijven.

## Wat is GroupDocs.Merger?

`GroupDocs.Merger` is een Java‑bibliotheek die PDF, DOCX, PPTX en vele andere documentformaten op de serverkant samenvoegt, splitst en bewerkt. Het werkt zonder dat Microsoft Office of Adobe Acrobat vereist is, waardoor het ideaal is voor backend‑services. Het ondersteunt het samenvoegen van documenten van verschillende formaten, behoudt de oorspronkelijke opmaak, metadata en annotaties, en kan met minimale configuratie worden geïntegreerd in Spring Boot‑ of gewone Java‑applicaties.

## Waarom GroupDocs-licentie Java instellen?

Het instellen van een licentie ontgrendelt **100 % van de API**, verwijdert de 20‑pagina evaluatiewatermark, en maakt verwerking van documenten tot **2 GB** in grootte mogelijk. In de niet‑gelicentieerde modus ben je beperkt tot 20 pagina's per document en wordt een zichtbare watermark toegevoegd, wat geautomatiseerde workflows kan verstoren.

## Vereisten

- Java 8 of hoger (Java 11+ aanbevolen)  
- Maven of Gradle build‑systeem  
- Een geldig GroupDocs.Merger‑licentiebestand (`.lic`) of stream, of een metered‑licentiesleutel van je GroupDocs‑account
