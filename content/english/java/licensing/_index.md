# Set GroupDocs License Java – Licensing Tutorials for GroupDocs.Merger

In this hub you’ll **set groupdocs license java** quickly and correctly, whether you work with a license file, an `InputStream`, or a metered‑usage model. We walk you through each licensing scenario, explain why proper licensing matters for production stability, and point you to the most up‑to‑date code samples. By following these guides you’ll avoid common pitfalls such as “License not found” errors and ensure your document‑processing pipelines stay fully functional.

## What is GroupDocs.Merger?

`GroupDocs.Merger` is a Java library that merges, splits, and manipulates PDF, DOCX, PPTX, and many other document formats on the server side. It operates without requiring Microsoft Office or Adobe Acrobat, making it ideal for backend services. It supports merging documents of different formats, preserving original formatting, metadata, and annotations, and can be integrated into Spring Boot or plain Java applications with minimal configuration.

## Why set GroupDocs license Java?

Setting a license unlocks **100 % of the API**, removes the 20‑page evaluation watermark, and enables processing of documents up to **2 GB** in size. Unlicensed mode limits you to 20 pages per document and adds a visible watermark, which can break automated workflows.

## Prerequisites

- Java 8 or later (Java 11+ recommended)  
- Maven or Gradle build system  
- A valid GroupDocs.Merger license file (`.lic`) or stream, or a metered‑license key from your GroupDocs account
