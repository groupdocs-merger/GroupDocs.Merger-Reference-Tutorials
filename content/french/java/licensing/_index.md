# Définir la licence GroupDocs Java – Tutoriels de licence pour GroupDocs.Merger

Dans ce hub, vous **définirez la licence groupdocs java** rapidement et correctement, que vous travailliez avec un fichier de licence, un `InputStream` ou un modèle d’utilisation à la consommation. Nous vous guidons à travers chaque scénario de licence, expliquons pourquoi une licence appropriée est essentielle pour la stabilité en production, et vous indiquons les exemples de code les plus à jour. En suivant ces guides, vous éviterez les pièges courants tels que les erreurs « Licence non trouvée » et vous assurerez que vos pipelines de traitement de documents restent pleinement fonctionnels.

## Qu’est‑ce que GroupDocs.Merger ?

`GroupDocs.Merger` est une bibliothèque Java qui fusionne, divise et manipule les formats PDF, DOCX, PPTX et de nombreux autres formats de documents côté serveur. Elle fonctionne sans nécessiter Microsoft Office ou Adobe Acrobat, ce qui la rend idéale pour les services backend. Elle prend en charge la fusion de documents de formats différents, en préservant la mise en forme originale, les métadonnées et les annotations, et peut être intégrée à des applications Spring Boot ou Java classiques avec une configuration minimale.

## Pourquoi définir la licence GroupDocs Java ?

Définir une licence débloque **100 % de l'API**, supprime le filigrane d'évaluation de 20 pages et permet le traitement de documents jusqu'à **2 GB** de taille. Le mode non licencié vous limite à 20 pages par document et ajoute un filigrane visible, ce qui peut perturber les flux de travail automatisés.

## Prérequis

- Java 8 ou version ultérieure (Java 11+ recommandé)  
- Système de construction Maven ou Gradle  
- Un fichier de licence GroupDocs.Merger valide (`.lic`) ou un flux, ou une clé de licence à la consommation depuis votre compte GroupDocs
