# Imposta licenza GroupDocs Java – Tutorial di licenza per GroupDocs.Merger

In questo hub **imposterai la licenza GroupDocs Java** rapidamente e correttamente, sia che tu lavori con un file di licenza, un `InputStream` o un modello a consumo. Ti guidiamo attraverso ogni scenario di licenza, spieghiamo perché una licenza corretta è importante per la stabilità in produzione e ti indirizziamo ai campioni di codice più aggiornati. Seguendo queste guide eviterai problemi comuni come errori “License not found” e garantirai che le tue pipeline di elaborazione documenti rimangano pienamente operative.

## Cos'è GroupDocs.Merger?

`GroupDocs.Merger` è una libreria Java che unisce, divide e manipola PDF, DOCX, PPTX e molti altri formati di documento sul lato server. Funziona senza richiedere Microsoft Office o Adobe Acrobat, rendendola ideale per servizi backend. Supporta l'unione di documenti di formati diversi, preservando la formattazione originale, i metadati e le annotazioni, e può essere integrata in applicazioni Spring Boot o Java plain con configurazione minima.

## Perché impostare la licenza GroupDocs Java?

Impostare una licenza sblocca **100 % dell'API**, rimuove la filigrana di valutazione di 20 pagine e consente l'elaborazione di documenti fino a **2 GB** di dimensione. La modalità non licenziata ti limita a 20 pagine per documento e aggiunge una filigrana visibile, che può interrompere i flussi di lavoro automatizzati.

## Prerequisiti

- Java 8 o successivo (consigliato Java 11+)  
- Sistema di build Maven o Gradle  
- Un file di licenza GroupDocs.Merger valido (`.lic`) o stream, oppure una chiave di licenza a consumo dal tuo account GroupDocs
