# GroupDocs.Merger ドキュメントマージ API – マージと分割の完全ガイド

**GroupDocs.Merger ドキュメントマージ API** は、.NET または Java アプリケーションから直接、さまざまなドキュメントタイプを高速かつクロスプラットフォームで結合、分割、操作できる方法を提供します。複数の PDF を 1 つのレポートにマージしたり、大きな契約書を個別のページに分割したり、プレゼンテーションのスライド順序を変更したりする場合でも、この API は数行のコードで対応できます。

## クイック回答
- **API は何をしますか？** 50 以上のフォーマットにわたってドキュメントをマージ、分割、順序変更、保護します。  
- **サポートされているプラットフォームは？** .NET (Standard, Framework, Core, Mono) と Java (J2SE 7‑10)。  
- **外部ソフトウェアは必要ですか？** いいえ — Microsoft Office や Adobe の依存関係はありません。  
- **ドキュメントを保護できますか？** はい、パスワードや権限の追加、削除、更新が可能です。  
- **ファイルサイズに制限はありますか？** この API は大容量ファイルやバッチ処理に最適化されています。  

## GroupDocs.Merger ドキュメントマージ API とは？
GroupDocs.Merger ドキュメントマージ API は、開発者向けに設計されたライブラリで、複数のドキュメント形式を扱う複雑さを抽象化します。プログラムからファイルをマージしたり、ページや範囲で分割したり、詳細なページ操作を適用したりでき、元のレイアウトやメタデータを保持します。

## なぜ GroupDocs.Merger ドキュメントマージ API を選ぶのか？
- **クロスプラットフォームの一貫性:** 1 回書けば Windows、Linux、macOS で動作します。  
- **幅広いフォーマットサポート:** PDF、Word、Excel、PowerPoint、画像、図表、アーカイブなど多数に対応。  
- **外部依存なし:** Office や Adobe のインストールが不要で、デプロイがシンプルです。  
- **高性能:** 大規模ドキュメントやバルク処理向けに設計され、メモリ使用量を削減します。  
- **組み込みセキュリティ:** パスワードの追加・削除や権限設定が簡単に行えます。

## 複数プラットフォームでのドキュメント管理を変革する
GroupDocs.Merger は、開発者がアプリケーション内でファイルを扱う方法を効率化する強力なクロスプラットフォームソリューションです。包括的な API により、複数のドキュメントを 1 つにマージしたり、ドキュメントを小さなコンポーネントに分割したり、ページ順序を変更したり、セキュリティオプションを適用したり、さまざまな操作を最小限のコードで実現できます。

.NET でも Java でも、GroupDocs.Merger は PDF、Office 文書、画像など 50 以上の一般的なファイル形式に対して、一貫した高性能なドキュメント処理機能を提供します。

## 主な機能と特長
- **包括的なドキュメントマージ** – 複数ソースから全体ドキュメントまたは特定ページを 1 つの統合ファイルに結合。  
- **クロスフォーマットサポート** – 異なるファイル形式 (例: DOC、XLSX、画像) を単一の PDF に統合。  
- **スマートドキュメント分割** – ページ番号、範囲、または特定ページを別ファイルとして抽出。  
- **高度なページ操作** – ページの移動、削除、回転、入れ替え、抽出を正確に制御。  
- **セキュリティ管理** – ドキュメントのパスワードや権限の追加・削除・更新。  
- **OLE オブジェクトサポート** – Word、Excel、PowerPoint、Visio ファイル内にドキュメントを OLE オブジェクトとして挿入。  
- **外部依存なし** – Microsoft Office や Adobe ソフトウェアを必要とせずに処理。  
- **高性能** – 大規模ドキュメントやバッチ処理を効率的に実行できるよう最適化。

## サポートされているファイル形式
GroupDocs.Merger は、複数のカテゴリにわたる 50 以上のファイル形式に対応しています。

- **ドキュメント**: PDF, DOC, DOCX, RTF, ODT, XPS, EPUB, HTML  
- **スプレッドシート**: XLS, XLSX, XLSM, XLSB, ODS, CSV, TSV  
- **プレゼンテーション**: PPT, PPTX, PPS, PPSX, ODP  
- **画像**: JPG, PNG, BMP, GIF, TIFF, SVG  
- **図表**: VSDX, VSDM, VSSX, VSTX, VDX, VSX, VTX  
- **アーカイブ**: ZIP, TAR, 7Z  
- **その他多数**

## よくある質問

**Q: 商用製品で API を使用できますか？**  
A: はい、有効な GroupDocs ライセンスがあれば使用可能です。評価用の無料トライアルも提供しています。

**Q: API はパスワード保護された PDF をサポートしていますか？**  
A: もちろんです。同じ API 呼び出しで暗号化された PDF を開き、変更し、再度保護できます。

**Q: .NET と Java のどのバージョンが互換性がありますか？**  
A: .NET Standard 2.0、.NET Framework 2.0+、.NET Core 2.0+、Mono 1.2+；Java 7、8、10+ が完全にサポートされています。

**Q: 非常に大きなファイルはどのように処理されますか？**  
A: データをストリーミングし、メモリ効率の高いアルゴリズムを使用するため、マルチギガバイトのドキュメントでもシステムリソースを枯渇させずに処理できます。

**Q: OCR やテキスト抽出の組み込みサポートはありますか？**  
A: コアのマージ API は構造操作に焦点を当てていますが、GroupDocs.Parser と組み合わせることで高度なテキスト抽出や OCR が可能です。

**Q: すぐに始めるにはどうすればよいですか？**  
A: .NET 用に NuGet パッケージ (`GroupDocs.Merger`) をインストールするか、Java 用に Maven 依存関係を追加し、まずは「ドキュメント読み込み」チュートリアルで最初のファイルをロードしてください。

**Q: 異なるファイルタイプを 1 つの PDF にマージできますか？**  
A: はい — API は PDF、Word、Excel、画像など多数のフォーマットを単一の PDF 出力にマージできます。
