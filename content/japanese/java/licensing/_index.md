# GroupDocs ライセンス Java の設定 – GroupDocs.Merger のライセンスチュートリアル

このハブでは、ライセンスファイル、`InputStream`、または従量課金モデルのいずれを使用する場合でも、**GroupDocs ライセンス Java を迅速かつ正確に設定**できます。各ライセンスシナリオを順に解説し、適切なライセンスが本番環境の安定性にとって重要な理由を説明し、最新のコードサンプルへ案内します。これらのガイドに従うことで、「License not found」エラーなどの一般的な落とし穴を回避し、ドキュメント処理パイプラインが完全に機能し続けることを保証します。

## GroupDocs.Mergerとは何ですか？

`GroupDocs.Merger` は、サーバーサイドで PDF、DOCX、PPTX など多数のドキュメント形式を結合、分割、操作できる Java ライブラリです。Microsoft Office や Adobe Acrobat を必要とせずに動作するため、バックエンドサービスに最適です。異なる形式のドキュメントを結合でき、元の書式、メタデータ、注釈を保持します。また、Spring Boot やプレーンな Java アプリケーションに最小限の設定で統合可能です。

## なぜ GroupDocs ライセンス Java を設定するのですか？

ライセンスを設定すると **API の 100 % が解放** され、20 ページの評価版ウォーターマークが除去され、最大 **2 GB** のサイズのドキュメントを処理できるようになります。未ライセンスモードではドキュメントあたり 20 ページに制限され、目に見えるウォーターマークが付加されるため、自動化ワークフローが壊れる可能性があります。

## 前提条件

- Java 8 以上（Java 11+ 推奨）  
- Maven または Gradle ビルドシステム  
- 有効な GroupDocs.Merger ライセンスファイル（`.lic`）またはストリーム、あるいは GroupDocs アカウントから取得した従量課金ライセンスキー
