**GroupDocs.Merger document merging API** zapewnia szybki, wieloplatformowy sposób łączenia, dzielenia i manipulacji szeroką gamą typów dokumentów bezpośrednio z aplikacji .NET lub Java. Niezależnie od tego, czy musisz scalić kilka plików PDF w jeden raport, podzielić duży kontrakt na poszczególne strony, czy zmienić kolejność slajdów w prezentacji, to API radzi sobie z tym przy użyciu zaledwie kilku linii kodu.

## Szybkie odpowiedzi
- **Co robi API?** Łączy, dzieli, zmienia kolejność i zabezpiecza dokumenty w ponad 50 formatach.  
- **Jakie platformy są obsługiwane?** .NET (Standard, Framework, Core, Mono) i Java (J2SE 7‑10).  
- **Czy potrzebuję zewnętrznego oprogramowania?** Nie — nie wymaga Microsoft Office ani zależności Adobe.  
- **Czy mogę zabezpieczyć dokumenty?** Tak, możesz dodawać, usuwać lub aktualizować hasła i uprawnienia.  
- **Czy istnieje limit rozmiaru pliku?** API jest zoptymalizowane pod kątem dużych plików i operacji wsadowych.  

## Co to jest GroupDocs.Merger document merging API?
GroupDocs.Merger document merging API to biblioteka skierowana do programistów, która ukrywa złożoność obsługi wielu formatów dokumentów. Umożliwia programowe scalanie plików, dzielenie ich według stron lub zakresów oraz stosowanie zaawansowanych operacji na stronach — wszystko przy zachowaniu oryginalnego układu i metadanych.

## Dlaczego wybrać GroupDocs.Merger document merging API?
- **Spójność wieloplatformowa:** Napisz raz, uruchom na Windows, Linux lub macOS.  
- **Szerokie wsparcie formatów:** Pracuj z PDF‑ami, dokumentami Word, Excel, PowerPoint, obrazami, diagramami, archiwami i innymi.  
- **Brak zewnętrznych zależności:** Nie wymaga instalacji Office ani Adobe, co upraszcza wdrożenie.  
- **Wysoka wydajność:** Zaprojektowane do obsługi dużych dokumentów i przetwarzania wsadowego, zmniejszając zużycie pamięci.  
- **Wbudowane zabezpieczenia:** Łatwe dodawanie i usuwanie haseł oraz ustawianie uprawnień.

## Zmieniaj sposób zarządzania dokumentami na wielu platformach
GroupDocs.Merger zapewnia potężne, wieloplatformowe rozwiązanie do manipulacji dokumentami, które usprawnia pracę programistów z plikami w ich aplikacjach. Nasze kompleksowe API umożliwia scalanie wielu dokumentów w jeden, dzielenie dokumentów na mniejsze części, zmianę kolejności stron, stosowanie opcji zabezpieczeń i wiele więcej — przy minimalnej ilości kodu i maksymalnej wydajności.

Niezależnie od tego, czy tworzysz aplikacje w środowiskach .NET czy Java, GroupDocs.Merger dostarcza spójne, wysokowydajne możliwości przetwarzania dokumentów, które działają bezproblemowo z ponad 50 popularnymi formatami plików, w tym PDF, dokumentami Office, obrazami i innymi.

## Kluczowe funkcje i możliwości
- **Kompleksowe scalanie dokumentów** – Łącz całe dokumenty lub wybrane strony z wielu źródeł w jeden spójny plik.  
- **Wsparcie wielu formatów** – Scalaj różne formaty plików w jeden dokument (np. DOC, XLSX i obrazy w pojedynczy PDF).  
- **Inteligentne dzielenie dokumentów** – Dziel dokumenty według numerów stron, zakresów lub wyodrębniaj wybrane strony jako osobne pliki.  
- **Zaawansowane operacje na stronach** – Przemieszczaj, usuwaj, obracaj, zamieniaj i wyodrębniaj strony z precyzyjną kontrolą.  
- **Zarządzanie zabezpieczeniami** – Dodawaj, usuwaj lub aktualizuj hasła i uprawnienia dokumentów.  
- **Wsparcie obiektów OLE** – Wstawiaj dokumenty jako obiekty OLE w plikach Word, Excel, PowerPoint i Visio.  
- **Brak zewnętrznych zależności** – Przetwarzaj dokumenty bez wymogu posiadania oprogramowania Microsoft Office lub Adobe.  
- **Wysoka wydajność** – Optymalizowane pod kątem efektywnego przetwarzania dużych dokumentów i operacji wsadowych.

## Obsługiwane formaty plików
GroupDocs.Merger obsługuje ponad 50 formatów plików w różnych kategoriach:

- **Dokumenty**: PDF, DOC, DOCX, RTF, ODT, XPS, EPUB, HTML  
- **Arkusze kalkulacyjne**: XLS, XLSX, XLSM, XLSB, ODS, CSV, TSV  
- **Prezentacje**: PPT, PPTX, PPS, PPSX, ODP  
- **Obrazy**: JPG, PNG, BMP, GIF, TIFF, SVG  
- **Diagramy**: VSDX, VSDM, VSSX, VSTX, VDX, VSX, VTX  
- **Archiwa**: ZIP, TAR, 7Z  
- **I wiele innych**

## Najczęściej zadawane pytania

**P: Czy mogę używać API w produkcie komercyjnym?**  
O: Tak, przy ważnej licencji GroupDocs. Dostępna jest bezpłatna wersja próbna do oceny.

**P: Czy API obsługuje PDF‑y zabezpieczone hasłem?**  
O: Zdecydowanie. Możesz otwierać, modyfikować i ponownie zabezpieczać zaszyfrowane PDF‑y przy użyciu tych samych wywołań API.

**P: Które wersje .NET i Java są kompatybilne?**  
O: .NET Standard 2.0, .NET Framework 2.0+, .NET Core 2.0+, Mono 1.2+; Java 7, 8 i 10+ są w pełni wspierane.

**P: Jak API radzi sobie z bardzo dużymi plikami?**  
O: Przesyła dane strumieniowo i używa algorytmów oszczędzających pamięć, co pozwala przetwarzać dokumenty o rozmiarze kilku gigabajtów bez wyczerpania zasobów systemowych.

**P: Czy istnieje wbudowane wsparcie dla OCR lub wyodrębniania tekstu?**  
O: Choć podstawowe API scalania koncentruje się na operacjach strukturalnych, możesz połączyć je z GroupDocs.Parser w celu zaawansowanego wyodrębniania tekstu i OCR.

**P: Jak szybko rozpocząć?**  
O: Zainstaluj pakiet NuGet (`GroupDocs.Merger`) dla .NET lub dodaj zależność Maven dla Java, a następnie przejdź do samouczka „Document Loading”, aby wczytać swój pierwszy plik.

**P: Czy mogę scalać różne typy plików w jeden PDF?**  
O: Tak — API może scalać PDF‑y, Word, Excel, obrazy i wiele innych formatów w jeden plik PDF.
