# Definir Licença GroupDocs Java – Tutoriais de Licenciamento para GroupDocs.Merger

Neste hub você **definirá a licença GroupDocs Java** rapidamente e corretamente, seja trabalhando com um arquivo de licença, um `InputStream` ou um modelo de uso medido. Nós guiamos você por cada cenário de licenciamento, explicamos por que o licenciamento adequado é importante para a estabilidade de produção e apontamos para os exemplos de código mais atualizados. Seguindo estes guias, você evitará armadilhas comuns, como erros “License not found”, e garantirá que seus pipelines de processamento de documentos permaneçam totalmente funcionais.

## O que é o GroupDocs.Merger?

`GroupDocs.Merger` é uma biblioteca Java que mescla, divide e manipula PDF, DOCX, PPTX e muitos outros formatos de documento no lado do servidor. Ela funciona sem exigir Microsoft Office ou Adobe Acrobat, tornando-a ideal para serviços de backend. Suporta a mesclagem de documentos de diferentes formatos, preservando a formatação original, metadados e anotações, e pode ser integrada ao Spring Boot ou a aplicações Java simples com configuração mínima.

## Por que definir a licença GroupDocs Java?

Definir uma licença desbloqueia **100 % da API**, remove a marca d'água de avaliação de 20 páginas e permite o processamento de documentos de até **2 GB** de tamanho. O modo sem licença limita você a 20 páginas por documento e adiciona uma marca d'água visível, o que pode interromper fluxos de trabalho automatizados.

## Pré-requisitos

- Java 8 ou superior (Java 11+ recomendado)  
- Sistema de build Maven ou Gradle  
- Um arquivo de licença válido do GroupDocs.Merger (`.lic`) ou stream, ou uma chave de licença medida da sua conta GroupDocs
