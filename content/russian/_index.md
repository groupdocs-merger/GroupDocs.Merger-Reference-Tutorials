Now produce final content.# GroupDocs.Merger Document Merging API – Полное руководство по объединению и разбиению

The **GroupDocs.Merger document merging API** предоставляет быстрый, кросс‑платформенный способ объединять, разбивать и манипулировать широким спектром типов документов непосредственно из ваших приложений на .NET или Java. Независимо от того, нужно ли вам объединить несколько PDF в один отчет, разбить большой контракт на отдельные страницы или переупорядочить слайды в презентации, этот API справится с задачей всего несколькими строками кода.

## Быстрые ответы
- **Что делает API?** Он объединяет, разбивает, переупорядочивает и защищает документы более чем в 50 форматах.  
- **Какие платформы поддерживаются?** .NET (Standard, Framework, Core, Mono) и Java (J2SE 7‑10).  
- **Нужен ли внешний софт?** Нет — никаких зависимостей от Microsoft Office или Adobe.  
- **Могу ли я защищать документы?** Да, вы можете добавлять, удалять или обновлять пароли и разрешения.  
- **Есть ли ограничение на размер файла?** API оптимизирован для работы с большими файлами и пакетными операциями.  

## Что такое GroupDocs.Merger document merging API?
GroupDocs.Merger document merging API — это библиотека, ориентированная на разработчиков, которая абстрагирует сложность работы с множеством форматов документов. Она позволяет программно объединять файлы, разбивать их по страницам или диапазонам и выполнять продвинутые операции со страницами — всё при сохранении оригинального макета и метаданных.

## Почему стоит выбрать GroupDocs.Merger document merging API?
- **Кросс‑платформенная согласованность:** Пишете один раз, запускаете на Windows, Linux или macOS.  
- **Широкая поддержка форматов:** Работа с PDF, Word, Excel, PowerPoint, изображениями, диаграммами, архивами и др.  
- **Отсутствие внешних зависимостей:** Не требуется установка Office или Adobe, что упрощает развертывание.  
- **Высокая производительность:** Разработан для больших документов и пакетной обработки, снижая потребление памяти.  
- **Встроенная безопасность:** Легко добавлять или удалять пароли и задавать разрешения.

## Преобразуйте способ управления документами на разных платформах
GroupDocs.Merger предоставляет мощное кросс‑платформенное решение для манипуляций с документами, упрощающее работу разработчиков с файлами в их приложениях. Наш всесторонний API позволяет объединять несколько документов в один, разбивать документы на более мелкие части, переупорядочивать страницы, применять параметры безопасности и многое другое — всё с минимальным объёмом кода и максимальной эффективностью.

Независимо от того, разрабатываете ли вы в среде .NET или Java, GroupDocs.Merger обеспечивает согласованные, высокопроизводительные возможности обработки документов, которые без проблем работают с более чем 50 популярными форматами файлов, включая PDF, офисные документы, изображения и др.

## Ключевые функции и возможности
- **Полное объединение документов** – Объединяйте целые документы или отдельные страницы из нескольких источников в один связный файл.  
- **Поддержка кросс‑форматов** – Объединяйте разные форматы файлов в единый документ (например, DOC, XLSX и изображения в один PDF).  
- **Умное разбиение документов** – Делите документы по номерам страниц, диапазонам или извлекайте отдельные страницы в отдельные файлы.  
- **Продвинутые операции со страницами** – Перемещайте, удаляйте, вращайте, меняйте местами и извлекайте страницы с точным контролем.  
- **Управление безопасностью** – Добавляйте, удаляйте или обновляйте пароли и разрешения документов.  
- **Поддержка OLE‑объектов** – Вставляйте документы как OLE‑объекты в файлы Word, Excel, PowerPoint и Visio.  
- **Отсутствие внешних зависимостей** – Обрабатывайте документы без необходимости установки Microsoft Office или Adobe.  
- **Высокая производительность** – Оптимизировано для эффективной обработки больших документов и пакетных операций.

## Поддерживаемые форматы файлов
GroupDocs.Merger работает более чем с 50 форматами файлов в различных категориях:
- **Документы**: PDF, DOC, DOCX, RTF, ODT, XPS, EPUB, HTML
- **Электронные таблицы**: XLS, XLSX, XLSM, XLSB, ODS, CSV, TSV
- **Презентации**: PPT, PPTX, PPS, PPSX, ODP
- **Изображения**: JPG, PNG, BMP, GIF, TIFF, SVG
- **Диаграммы**: VSDX, VSDM, VSSX, VSTX, VDX, VSX, VTX
- **Архивы**: ZIP, TAR, 7Z
- **И многое другое**

## Часто задаваемые вопросы

**Q: Можно ли использовать API в коммерческом продукте?**  
A: Да, при наличии действующей лицензии GroupDocs. Доступна бесплатная пробная версия для оценки.

**Q: Поддерживает ли API PDF с паролем?**  
A: Абсолютно. Вы можете открывать, изменять и повторно защищать зашифрованные PDF, используя те же вызовы API.

**Q: Какие версии .NET и Java совместимы?**  
A: .NET Standard 2.0, .NET Framework 2.0+, .NET Core 2.0+, Mono 1.2+; Java 7, 8 и 10+ полностью поддерживаются.

**Q: Как API обрабатывает очень большие файлы?**  
A: Он потоково передаёт данные и использует алгоритмы с низким потреблением памяти, позволяя обрабатывать многогигабайтные документы без исчерпания ресурсов системы.

**Q: Есть ли встроенная поддержка OCR или извлечения текста?**  
A: Хотя основной API объединения сосредоточен на структурных операциях, вы можете комбинировать его с GroupDocs.Parser для продвинутого извлечения текста и OCR.

**Q: Как быстро начать работу?**  
A: Установите пакет NuGet (`GroupDocs.Merger`) для .NET или добавьте зависимость Maven для Java, затем следуйте учебнику «Загрузка документов», чтобы загрузить ваш первый файл.

**Q: Можно ли объединять разные типы файлов в один PDF?**  
A: Да — API может объединять PDF, Word, Excel, изображения и многие другие форматы в один PDF‑файл.
