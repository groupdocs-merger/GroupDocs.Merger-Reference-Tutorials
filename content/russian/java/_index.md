## Как разделить PDF в Java?

Разделите PDF на отдельные страницы или пользовательские диапазоны всего в три строки Java‑кода. Вызовите метод `split()` у экземпляра `Merger`, передайте путь к исходному файлу и укажите желаемый диапазон страниц или размер. Библиотека записывает каждый сегмент в отдельный файл, сохраняя оригинальное качество и метаданные.

Можно также разделять по размеру (например, части по 5 МБ) или по списку номеров страниц, что идеально подходит для создания PDF‑глав из единого мастер‑документа. Операция выполняется за линейное время относительно количества страниц, что делает её пригодной для масштабной пакетной обработки.

## Как объединить несколько PDF в Java?

Загрузите каждый исходный PDF с помощью метода `addDocument()` класса `Merger`, расположите их в нужном порядке и вызовите `merge()`. API автоматически согласует размеры страниц, обновит закладки и объединит свойства документов. Вы также можете объединять PDF с другими форматами — например, Word или Excel — конвертируя их «на лету», получая единый PDF‑вывод.

Для сценариев с высоким пропускным способностью включите режим потоковой передачи, чтобы избежать загрузки целых файлов в память. Это снижает использование кучи до 80 % при обработке документов со сотнями страниц.

## Понимание класса Merger

Класс `Merger` — основной компонент GroupDocs.Merger for Java, который координирует объединение, разделение и операции безопасности над документами. Он предоставляет цепочечные методы, такие как `addDocument()`, `split()`, `protect()` и `merge()`, позволяющие строить лаконичные конвейеры обработки.

### Обзор ключевых методов
- `addDocument(String path)`: Добавляет исходный файл в очередь для последующего объединения.  
- `split(String source, SplitOptions options)`: Делит документ на основе диапазонов страниц или ограничений по размеру.  
- `protect(String output, ProtectionOptions options)`: Применяет парольную защиту и ограничения разрешений.  
- `merge(String output)`: Выполняет все запланированные операции и записывает окончательный документ.

Эти методы потокобезопасны и могут быть объединены в цепочку для выразительного, читаемого кода.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|----------|
| **Ошибки «Out‑of‑memory» при работе с большими PDF** | Загрузка полного файла в память | Включите режим потоковой передачи (`Merger.setStreaming(true)`) или разделите файл на более мелкие части перед объединением. |
| **Несоответствие ориентации страниц** | Исходные PDF содержат смешанные портретные и альбомные страницы | Используйте `rotatePage(int pageNumber, RotationAngle angle)` перед объединением, чтобы стандартизировать ориентацию. |
| **Не удаётся открыть защищённый паролем источник** | Отсутствует пароль дешифрования | Укажите пароль через `DocumentLoadOptions.setPassword("yourPwd")` при добавлении документа. |
| **Метаданные теряются после объединения** | По умолчанию объединение отбрасывает пользовательские метаданные | Вызовите `setPreserveMetadata(true)` у экземпляра `Merger`, чтобы сохранить оригинальные свойства. |

## Часто задаваемые вопросы

**В: Как разделить PDF с помощью GroupDocs.Merger в Java?**  
О: Создайте экземпляр `Merger`, вызовите `split(sourcePath, new SplitOptions().setPageRanges("1-3,5,7-10"))` и укажите папку вывода — каждый диапазон станет отдельным PDF‑файлом.

**В: Можно ли объединять PDF и Excel‑таблицы вместе?**  
О: Да — GroupDocs.Merger поддерживает кросс‑форматное объединение, позволяя комбинировать PDF с Excel‑файлами (`merge excel files java`) в один PDF‑вывод.

**В: Как добавить парольную защиту после объединения?**  
О: После вызова `merge()` выполните `protect(outputPath, new ProtectionOptions().setPassword("Secret123"))`, чтобы зашифровать финальный документ.

**В: Возможно ли объединять PDF без загрузки всего файла в память?**  
О: Включите потоковую передачу (`Merger.setStreaming(true)`), чтобы обрабатывать файлы буферно, что значительно снижает потребление памяти для больших документов.

**В: Какая лицензия требуется для продакшн‑использования?**  
О: Для продакшн‑развёртываний обязательна коммерческая лицензия GroupDocs.Merger; бесплатная 30‑дневная пробная версия доступна для разработки и тестирования.
