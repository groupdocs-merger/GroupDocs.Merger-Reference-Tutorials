# Configurar la licencia de GroupDocs Java – Tutoriales de licenciamiento para GroupDocs.Merger

En este hub **configurarás la licencia de groupdocs java** de forma rápida y correcta, ya sea que trabajes con un archivo de licencia, un `InputStream` o un modelo de uso medido. Te guiamos a través de cada escenario de licenciamiento, explicamos por qué una licencia adecuada es importante para la estabilidad en producción y te señalamos los ejemplos de código más actualizados. Al seguir estas guías evitarás problemas comunes como errores de “License not found” y garantizarás que tus flujos de procesamiento de documentos permanezcan totalmente funcionales.

## ¿Qué es GroupDocs.Merger?

`GroupDocs.Merger` es una biblioteca Java que fusiona, divide y manipula PDF, DOCX, PPTX y muchos otros formatos de documentos en el lado del servidor. Funciona sin requerir Microsoft Office o Adobe Acrobat, lo que la hace ideal para servicios backend. Soporta la fusión de documentos de diferentes formatos, preservando el formato original, los metadatos y las anotaciones, y puede integrarse en aplicaciones Spring Boot o Java simples con una configuración mínima.

## ¿Por qué configurar la licencia de GroupDocs Java?

Configurar una licencia desbloquea **100 % de la API**, elimina la marca de agua de evaluación de 20 páginas y permite procesar documentos de hasta **2 GB** de tamaño. El modo sin licencia te limita a 20 páginas por documento y añade una marca de agua visible, lo que puede romper flujos de trabajo automatizados.

## Requisitos previos

- Java 8 o posterior (se recomienda Java 11+)  
- Sistema de compilación Maven o Gradle  
- Un archivo de licencia válido de GroupDocs.Merger (`.lic`) o stream, o una clave de licencia medida de tu cuenta de GroupDocs
