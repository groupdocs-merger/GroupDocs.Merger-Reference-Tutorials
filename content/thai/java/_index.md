# วิธีแยก PDF และรวม PDF ด้วย GroupDocs.Merger สำหรับ Java

ในแอปพลิเคชัน Java สมัยใหม่, **split pdf java** เป็นความต้องการที่พบบ่อย—ไม่ว่าคุณจะต้องการแยกรายงานขนาดใหญ่เป็นบทย่อยหรือรวมใบแจ้งหนี้หลายฉบับเป็นไฟล์เดียว GroupDocs.Merger for Java ทำให้การแยกและการรวม PDF (และรูปแบบอื่นกว่า 50 รูปแบบ) ง่ายดาย ด้วยการประมวลผลที่มีประสิทธิภาพสูงเพียงไม่กี่บรรทัดของโค้ด ในบทแนะนำนี้เราจะสำรวจ API หลัก, ผ่านสถานการณ์จริง, และชี้แนะคุณไปยังบทแนะนำเชิงลึกสำหรับทุกความต้องการการประมวลผลเอกสารที่คุณอาจเจอ

## คำตอบอย่างรวดเร็ว
- **อะไรคือการใช้งานหลักของ GroupDocs.Merger?** รวม, แยก, และจัดการเอกสารในรูปแบบมากกว่า 50 รูปแบบ รวมถึง PDF, Word, Excel, และรูปภาพ.  
- **ฉันสามารถแยก PDF ใน Java ได้หรือไม่?** ใช่ – API มีเมธอด “split pdf java” เฉพาะที่ให้คุณกำหนดช่วงหน้า หรือการแยกตามขนาด.

## วิธีแยก PDF ด้วย Java?
แยก PDF เป็นหน้าเดี่ยวหรือช่วงที่กำหนดเองด้วยเพียงสามบรรทัดของ Java. เรียกเมธอด `split()` บนอินสแตนซ์ `Merger`, ส่งพาธไฟล์ต้นฉบับ, และระบุช่วงหน้าหรือขนาดที่ต้องการ. ไลบรารีจะเขียนแต่ละส่วนไปยังไฟล์แยกโดยคงคุณภาพและเมตาดาต้าต้นฉบับ.

คุณยังสามารถแยกตามขนาด (เช่น ชิ้นส่วน 5 MB) หรือโดยรายการหมายเลขหน้า, ซึ่งเหมาะสำหรับการสร้าง PDF แบ่งตามบทจากเอกสารหลักเดียว. การดำเนินการทำงานในเวลาเชิงเส้นตามจำนวนหน้า, ทำให้เหมาะกับการประมวลผลแบบแบตช์ขนาดใหญ่.

## วิธีรวม PDF หลายไฟล์ด้วย Java?
โหลด PDF ต้นฉบับแต่ละไฟล์ด้วยเมธอด `addDocument()` ของ `Merger`, จัดเรียงตามลำดับที่ต้องการ, แล้วเรียก `merge()`. API จะทำให้มิติหน้าตรงกันอัตโนมัติ, อัปเดตบุ๊กมาร์ก, และรวมคุณสมบัติของเอกสาร. คุณยังสามารถรวม PDF กับรูปแบบอื่น ๆ เช่น Word หรือ Excel โดยแปลงระหว่างการทำงาน, ทำให้ได้ผลลัพธ์ PDF เดียวที่เป็นเอกภาพ.

สำหรับสถานการณ์ที่ต้องการประมวลผลสูง, เปิดโหมดสตรีมมิ่งเพื่อหลีกเลี่ยงการโหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำ. วิธีนี้ลดการใช้ heap ได้ถึง 80 % เมื่อประมวลผลเอกสารที่มีหลายร้อยหน้า.

## ทำความเข้าใจคลาส Merger
คลาส `Merger` เป็นส่วนประกอบหลักของ GroupDocs.Merger for Java ที่จัดการการรวมเอกสาร, การแยก, และการดำเนินการด้านความปลอดภัย. มันเปิดเผยเมธอดแบบ fluent เช่น `addDocument()`, `split()`, `protect()`, และ `merge()` เพื่อสร้างสายการประมวลผลที่กระชับ.

### ภาพรวมเมธอดสำคัญ
- `addDocument(String path)`: คิวไฟล์ต้นฉบับเพื่อการรวมในภายหลัง.  
- `split(String source, SplitOptions options)`: แบ่งเอกสารตามช่วงหน้า หรือขีดจำกัดขนาด.  
- `protect(String output, ProtectionOptions options)`: ใช้การป้องกันด้วยรหัสผ่านและจำกัดสิทธิ์.  
- `merge(String output)`: ดำเนินการเมธอดที่คิวไว้และเขียนเอกสารสุดท้าย.  

เมธอดเหล่านี้ปลอดภัยต่อเธรดและสามารถต่อเชื่อมกันเพื่อโค้ดที่อ่านง่ายและแสดงออกได้.

## ปัญหาทั่วไปและวิธีแก้
| Issue | Cause | Solution |
|-------|-------|----------|
| **ข้อผิดพลาด Out‑of‑memory บน PDF ขนาดใหญ่** | โหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำ | เปิดโหมดสตรีมมิ่ง (`Merger.setStreaming(true)`) หรือแยกไฟล์เป็นชิ้นส่วนเล็กก่อนทำการรวม. |
| **ความไม่ตรงกันของการวางแนวหน้า** | PDF ต้นฉบับมีหน้าที่เป็นแนวตั้งและแนวนอนผสมกัน | ใช้ `rotatePage(int pageNumber, RotationAngle angle)` ก่อนทำการรวมเพื่อทำให้แนวหน้ามาตรฐาน. |
| **ไฟล์ต้นฉบับที่มีการป้องกันด้วยรหัสผ่านไม่สามารถเปิดได้** | ไม่มีรหัสผ่านถอดรหัส | ระบุรหัสผ่านผ่าน `DocumentLoadOptions.setPassword("yourPwd")` ขณะเพิ่มเอกสาร. |
| **เมตาดาต้าหายหลังการรวม** | การรวมแบบเริ่มต้นละทิ้งเมตาดาต้ากำหนดเอง | เรียก `setPreserveMetadata(true)` บนอินสแตนซ์ `Merger` เพื่อคงคุณสมบัติดั้งเดิม. |

## คำถามที่พบบ่อย
**ถาม: ฉันจะแยก PDF ด้วย GroupDocs.Merger ใน Java อย่างไร?**  
**ตอบ:** สร้างอินสแตนซ์ `Merger`, เรียก `split(sourcePath, new SplitOptions().setPageRanges("1-3,5,7-10"))`, และระบุโฟลเดอร์ผลลัพธ์—แต่ละช่วงจะกลายเป็นไฟล์ PDF แยกต่างหาก.

**ถาม: ฉันสามารถรวม PDF กับแผ่นงาน Excel เข้าด้วยกันได้หรือไม่?**  
**ตอบ:** ได้—GroupDocs.Merger รองรับการรวมข้ามรูปแบบ, ทำให้คุณสามารถรวม PDF กับไฟล์ Excel (`merge excel files java`) เป็นผลลัพธ์ PDF เดียวได้.

**ถาม: ฉันจะเพิ่มการป้องกันด้วยรหัสผ่านหลังการรวมได้อย่างไร?**  
**ตอบ:** หลังจากเรียก `merge()`, เรียก `protect(outputPath, new ProtectionOptions().setPassword("Secret123"))` เพื่อเข้ารหัสเอกสารสุดท้าย.

**ถาม: สามารถรวม PDF ได้โดยไม่โหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำหรือไม่?**  
**ตอบ:** เปิดสตรีมมิ่ง (`Merger.setStreaming(true)`) เพื่อประมวลผลไฟล์แบบบัฟเฟอร์, ซึ่งลดการใช้หน่วยความจำอย่างมากสำหรับเอกสารขนาดใหญ่.

**ถาม: ต้องการไลเซนส์แบบใดสำหรับการใช้งานในสภาพแวดล้อมการผลิต?**  
**ตอบ:** จำเป็นต้องมีไลเซนส์เชิงพาณิชย์ของ GroupDocs.Merger สำหรับการปรับใช้ในการผลิต; มีการทดลองใช้ฟรี 30 วันสำหรับการพัฒนาและทดสอบ.
