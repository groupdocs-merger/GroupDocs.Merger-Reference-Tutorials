## Cách thiết lập GroupDocs Merger cho Java?
Cài đặt thư viện qua Maven hoặc Gradle, sau đó tạo một thể hiện `Merger` trỏ tới một tệp RTF hiện có. **Merger** là lớp chính do GroupDocs Merger cung cấp để điều phối các thao tác hợp nhất tài liệu. Điều này thiết lập tài liệu cơ sở mà các tệp tiếp theo sẽ được nối vào.

```java
import com.groupdocs.merger.Merger;

public class InitializeMerger {
    public static void run() throws Exception {
        Merger merger = new Merger("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF");
    }
}
```  
Đoạn mã trên tải tệp RTF đầu tiên, chuẩn bị API cho các thao tác tiếp theo.

## Cách khởi tạo Merger với tài liệu nguồn?
Tải tệp RTF chính của bạn vào đối tượng `Merger`; đối tượng này trở thành container cho tất cả các lần nối tiếp theo. Lớp `Merger` quản lý hàng đợi hợp nhất và xử lý luồng nội dung tài liệu.

```java
import com.groupdocs.merger.Merger;

public class InitializeMerger {
    public static void run() throws Exception {
        // Load the source RTF file
        Merger merger = new Merger("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF");
    }
}
```  
- **Mục đích**: Đặt tài liệu cơ sở.  
- **Tham số**: Đường dẫn tới tệp RTF nguồn.

## Cách thêm tài liệu khác để hợp nhất?
Phương thức `join` thêm một tài liệu khác vào hàng đợi hợp nhất hiện tại. **join** nhận đường dẫn của tệp RTF bổ sung và thêm nó vào danh sách tệp trong bộ nhớ để được kết hợp.

```java
Merger merger = new Merger("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF");
```  

```java
merger.join("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF_2");
```  
- **Mục đích**: Thêm tệp RTF thứ hai vào tài liệu cơ sở.  
- **Tham số**: Đường dẫn của tệp RTF cần hợp nhất.

## Cách lưu tài liệu đã hợp nhất?
Phương thức `save` ghi nội dung đã kết hợp vào một tệp mới ở định dạng mong muốn. **save** nhận đường dẫn đích và tùy chọn định dạng đầu ra, hoàn thiện thao tác hợp nhất.

```java
Merger merger = new Merger("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF");
merger.join("YOUR_DOCUMENT_DIRECTORY/SAMPLE_RTF_2");
```  

```java
String outputFile = "YOUR_OUTPUT_DIRECTORY/merged.rtf";
merger.save(outputFile);
```  
- **Mục đích**: Ghi nội dung đã kết hợp vào một tệp RTF mới.  
- **Tham số**: Đường dẫn tệp đích.

2. **Tổng hợp dữ liệu nghiên cứu** – Tập hợp các bản thảo chương cho việc nộp tạp chí.

## Các yếu tố hiệu năng
- **Tối ưu hóa bộ nhớ**: API truyền dữ liệu theo luồng, vì vậy việc sử dụng bộ nhớ vẫn dưới **150 MB** ngay cả khi hợp nhất tài liệu 500 trang.  
- **Xử lý chia đoạn**: Đối với các tệp cực lớn, chia hợp nhất thành các phần logic và gọi `join` tuần tự.  
- **Cập nhật thường xuyên**: Sử dụng phiên bản thư viện mới nhất để hưởng lợi từ các bản vá hiệu năng và hỗ trợ định dạng mới.

## Các vấn đề thường gặp và giải pháp
- **OutOfMemoryError** – Tăng bộ nhớ heap JVM (`-Xmx2g`) hoặc xử lý các tệp theo lô nhỏ hơn.  
- **Mất định dạng** – Đảm bảo các tệp RTF nguồn sử dụng mã hóa tương thích; API bảo tồn bảng, hình ảnh và kiểu dáng khi các tệp được tạo đúng chuẩn.
